# RestAPI

アカウント情報全体について変更する場合は、put。
アカウントのステータスの変更をする場合は、patch。


## PUT

- リソース全体を置き換えるために使用されます。
- リクエストボディには、更新後のリソース全体のデータを含める必要があります。
- 同じリクエストを複数回送信しても、結果は同じになります (冪等性)。

## PATCH

- リソースの一部を更新するために使用されます。
- リクエストボディには、更新する部分のデータのみを含めることができます。
- 同じリクエストを複数回送信すると、結果が変わる可能性があります (冪等性がない)。

例えば、ユーザー情報を更新する API があるとします。

PUT
```
PUT /users/123
{
    "name": "John Doe",
    "email": "john.doe@example.com",
    "age": 30
}
```

この場合、ユーザー ID 123 のユーザー情報を、リクエストボディで指定されたデータで完全に置き換えます。

PATCH
```
PATCH /users/123
{
    "email": "new.email@example.com"
}
```

この場合、ユーザー ID 123 のユーザー情報の email だけを更新します。
